## 2. Продуктовое описание

### 2.1. Общее описание продукта

**TeleManga** -- веб-приложение для чтения манги, доступное через Telegram Mini App. Пользователь запускает приложение прямо из Telegram, без необходимости устанавливать отдельное приложение или переходить на внешний сайт.

Платформа позволяет:
- просматривать каталог манги с фильтрацией по жанрам и тегам;
- читать мангу постранично;
- оценивать мангу по 10-балльной шкале;
- оставлять комментарии;
- вести персональную библиотеку со статусами чтения;
- автоматически сохранять прогресс чтения.

### 2.2. Целевая аудитория

Пользователи мессенджера Telegram, интересующиеся чтением манги на мобильных устройствах и десктопе.

### 2.3. Пользовательские сценарии

#### 2.3.1. Регистрация и вход в приложение

**Первый запуск (регистрация):**

1. Пользователь открывает Telegram-бота или переходит по ссылке на Mini App.
2. Нажимает кнопку запуска Mini App.
3. Приложение открывается в WebView внутри Telegram.
4. Поскольку пользователь ещё не зарегистрирован, отображается форма регистрации с полями: username, email, password.
5. После заполнения и отправки формы Backend создаёт учётную запись и возвращает JWT-токен.
6. Пользователь попадает на главную страницу.

**Повторный вход (авторизация):**

1. Пользователь открывает Mini App.
2. Если JWT-токен ещё действителен -- вход происходит автоматически.
3. Если токен истёк, отображается форма входа с полями: username, password.
4. После успешной авторизации Backend возвращает новый JWT-токен.

**Блокировка бота:**

Если пользователь заблокировал Telegram-бота, Mini App становится недоступным через кнопку бота. В этом случае:

1. Пользователь может перейти в Mini App по прямой ссылке (t.me/<bot_username>/<app_name>), если она доступна.
2. Данные пользователя (библиотека, история чтения, комментарии, оценки) сохраняются в БД и не удаляются при блокировке бота.
3. При разблокировке бота пользователь может снова войти в Mini App через кнопку бота и продолжить работу с сохранёнными данными.

#### 2.3.2. Просмотр каталога

1. На главной странице пользователь видит каталог манги: обложки, названия, средние оценки, количество просмотров.
2. Пользователь может фильтровать каталог по жанрам и тегам.
3. Пользователь может искать мангу по названию.
4. Каталог поддерживает сортировку (по популярности, рейтингу, дате добавления).

#### 2.3.3. Просмотр карточки манги

1. Пользователь нажимает на мангу в каталоге.
2. Открывается страница с детальной информацией: обложка, название, автор, описание, статус публикации, дата выхода, жанры, теги, средний рейтинг, количество просмотров.
3. Ниже -- список глав, сгруппированных по томам (при наличии), с указанием количества страниц.
4. На этой же странице расположены блоки комментариев и оценки.

#### 2.3.4. Чтение манги

1. Пользователь выбирает главу из списка.
2. Открывается режим чтения -- последовательный просмотр страниц-изображений.
3. Поддерживается перелистывание вперёд/назад, переход к конкретной странице.
4. Прогресс чтения (номер последней просмотренной страницы и главы) сохраняется автоматически.
5. При повторном открытии манги пользователь может продолжить с того места, где остановился.

#### 2.3.5. Оценка манги

1. На странице манги пользователь выбирает оценку от 1 до 10.
2. Оценка сохраняется; средний рейтинг манги пересчитывается.
3. Пользователь может изменить свою оценку.

#### 2.3.6. Комментирование

1. На странице манги пользователь видит список комментариев других пользователей.
2. Пользователь может написать свой комментарий.
3. Пользователь может редактировать свои комментарии.

#### 2.3.7. Управление библиотекой

1. На странице манги пользователь может добавить мангу в свою библиотеку, выбрав статус (читаю, прочитано, в планах, брошено, избранное).
2. В разделе «Моя библиотека» отображается список добавленной манги, сгруппированный по статусам.
3. Пользователь может изменить статус или удалить мангу из библиотеки.

#### 2.3.8. Администрирование (для роли admin)

1. Администратор может добавлять новую мангу (название, описание, обложка, автор, статус, жанры, теги).
2. Администратор может добавлять главы к манге и загружать страницы (изображения) к главам.
3. Администратор может редактировать и удалять мангу, главы, страницы, жанры и теги.

### 2.4. Структура экранов

| Экран | Описание |
|-------|----------|
| **Главная / Каталог** | Список манги с обложками, поиск, фильтры по жанрам и тегам, сортировка |
| **Карточка манги** | Детальная информация о манге, список глав, комментарии, оценка, кнопка «Добавить в библиотеку» |
| **Ридер (Чтение)** | Полноэкранный просмотр страниц главы с навигацией |
| **Моя библиотека** | Список манги пользователя, сгруппированный по статусам чтения |
| **Профиль** | Информация о пользователе, аватар, статистика (сколько манги прочитано, сколько оценок поставлено) |
| **Админ-панель** | Управление каталогом: CRUD манги, глав, страниц, жанров, тегов (доступна только для admin) |
